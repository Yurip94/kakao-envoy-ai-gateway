# v0.4 Envoy Gateway values 경로 404

## 상태

- 상태: resolved
- 발견일: 2026-04-23
- 해결일: 2026-04-23
- 관련 Seed: Seed 11 - 실제 Kind 클러스터로 v0.4 baseline 검증

## 배경

v0.4 baseline 실제 검증에서 `deploy/gateway/v0.4/install.sh`를 실행해 Envoy Gateway를 설치하던 중 values 파일 URL 오류가 발생했다.

## 증상

실패한 단계:

```bash
deploy/gateway/v0.4/install.sh
```

오류:

```text
failed to fetch https://raw.githubusercontent.com/envoyproxy/ai-gateway/v0.4.0/manifests/envoy-gateway-config/config.yaml : 404 Not Found
```

## 영향

- Seed 11의 v0.4 baseline 설치가 중단됨
- Envoy Gateway 설치가 시작되지 못해 이후 AI Gateway CRD/controller 설치로 진행 불가

## 원인

v0.4 태그에서 values 파일 경로를 잘못 사용했다.

- 잘못된 경로: `manifests/envoy-gateway-config/config.yaml`
- 올바른 경로: `manifests/envoy-gateway-values.yaml`

## 해결

다음 파일을 수정했다.

- `deploy/gateway/v0.4/install.sh`
- `docs/v0.4-baseline-plan.md`

수정 내용:

- Envoy Gateway values URL을 `https://raw.githubusercontent.com/envoyproxy/ai-gateway/v0.4.0/manifests/envoy-gateway-values.yaml`로 변경

## 검증

확인 명령:

```bash
curl -sI https://raw.githubusercontent.com/envoyproxy/ai-gateway/v0.4.0/manifests/envoy-gateway-values.yaml | head -n 1
```

결과:

```text
HTTP/2 200
```

추가 확인:

```bash
curl -sI https://raw.githubusercontent.com/envoyproxy/ai-gateway/v0.4.0/manifests/envoy-gateway-config/config.yaml | head -n 1
```

결과:

```text
HTTP/2 404
```

## 남은 리스크

- v0.4 설치 단계에서 다른 chart 호환성 이슈가 있을 수 있다.
- 실제 설치 재시도 결과를 기준으로 후속 이슈 여부를 판단한다.
